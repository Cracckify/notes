# Crackify Handbook
## Authentication & Authorization

> Learn Authentication & Authorization by understanding **why** things exist, not by memorizing definitions.

---

## About this Handbook

This handbook is designed for:

- Software Engineers preparing for Backend Interviews
- Java & Spring Boot Developers
- Students learning Authentication & Authorization
- Developers who want to understand how modern applications secure APIs

Unlike traditional notes, this handbook focuses on **engineering concepts** before implementation.

Instead of asking:

> *"What is JWT?"*

we first ask:

> *"Why was JWT invented?"*

Understanding the problem first makes the solution much easier to remember.

---

## Prerequisites

You should be familiar with:

- Java Basics
- Object Oriented Programming
- HTTP & REST APIs
- Basic SQL
- Spring Boot (Beginner Level)

No prior knowledge of Spring Security or OAuth2 is required.

---

## Learning Objectives

After completing this handbook, you should be able to:

- Explain Authentication and Authorization confidently
- Understand password hashing and why BCrypt is used
- Compare Sessions and JWT authentication
- Explain OAuth2 and OpenID Connect
- Implement Spring Security in a Spring Boot application
- Design secure authentication for microservices
- Answer common interview questions on backend security

## How to Read This Book

The chapters are designed to be read in order.

Each chapter builds on concepts introduced earlier.

For example:

Password Security
↓
Sessions
↓
JWT
↓
OAuth2
↓
Spring Security

Skipping chapters is possible, but reading sequentially will provide a much stronger understanding.

Technology changes constantly.

Frameworks evolve.

Libraries become outdated.

Engineering principles, however, remain relevant.

This handbook focuses on understanding those principles.

Once you understand **why** a technology exists, learning **how** to use it becomes significantly easier.
